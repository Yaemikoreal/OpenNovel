# 📜 L.O.O.M. V1.0.1 技术实施方案 (Production-Grade Blueprint)

## 一、 核心架构与防爆哲学

L.O.O.M. 遵循三层解耦架构，并以**“零信任 AI 输出”**为底层设计原则：

1.  **Human Layer (创作层)**：纯 Markdown，永远可 Git 追踪，永远可被 Obsidian/VSCode 打开。L.O.O.M. 退出后，作品依然完整。
2.  **Machine Shadow (状态层)**：YAML (即时缓存) + SQLite (全局账本) + Snapshots (时间机器)。
3.  **Semantic Layer (检索层)**：LlamaIndex + BGE-M3，将历史变为可召回的活体结构。

**四条防爆铁律：**
*   **铁律 1：ID 即锚点**。拒绝使用易变自然语言作为系统关联符，全局强制约束 Canonical IDs。
*   **铁律 2：权威分级**。上下文不是平的，设定具有最高神格，灵感仅作参考。
*   **铁律 3：人工审核关口**。AI 只能提议状态变更，人类拥有绝对否决权。
*   **铁律 4：操作可逆**。任何破坏性状态写入，必须前置快照，支持一键回滚。

## 三、 数据层设计 (带防爆机制)

### 3.1 Canonical ID 规范
所有系统内部引用、YAML 关联、SQLite 外键，**严禁使用角色名**，必须使用稳定 ID。

```yaml
# characters/alice.md
---
id: char_001  # 绝对锚点
name: Alice   # 当前代号
aliases: [Alys, The Witch] # 历史曾用名/代号
---
```

在章节 Frontmatter 中：
```yaml
pov: char_001
active_characters: [char_001, char_002]
```

### 3.2 权威分级上下文
注入上下文时，必须打上不可磨灭的权威性标签，LLM 必须严格遵守冲突降级逻辑（Canon > State > Memory > Subconscious）。

```markdown
[CANON | IMMUTABLE | HIGH AUTHORITY]
魔法需要消耗寿命。绝对禁止无限施法。

[STATE MEMORY | MEDIUM AUTHORITY]
char_001 当前左臂骨折，无法挥舞重剑。

[SUBCONSCIOUS FRAGMENT | LOW AUTHORITY | OPTIONAL]
“深渊不收我，因为我就是深渊。” (仅作为文风参考，非既定事实)
```

### 3.3 快照与回滚机制
每次执行 `loom commit` 前，系统自动生成快照。

```text
.snapshots/
└── ch_001_1698765432.snapshot.json
```

快照内容：
```json
{
  "timestamp": "2023-10-31T10:00:00",
  "chapter": "ch_001",
  "frontmatter_before": {"pov": "char_001", "emotion_vector": {}},
  "frontmatter_after": {"pov": "char_001", "emotion_vector": {"grief": 0.8}},
  "events_added_to_sqlite": ["evt_001_injury"]
}
```

## 四、 核心引擎实现：上下文组装器 (带熔断与分级)

```python
# 伪代码：core/context_assembler.py
import tiktoken

def assemble_actor_context(chapter_path: str, current_text: str) -> list[dict]:
    BUDGET = 8000 # 预留 2000 给输出
    current_tokens = 0
    context = []
    
    # 1. 人格 (最高优先级)
    prompt = load_prompt("actor.v1.md")
    context.append({"role": "system", "content": prompt})
    current_tokens += count_tokens(prompt)

    # 2. 设定 (IMMUTABLE HIGH AUTHORITY)
    canon = vector_retriever.query("当前场景规则", top_k=3)
    canon_text = f"[CANON | IMMUTABLE]\n{canon}"
    context.append({"role": "system", "content": canon_text})
    current_tokens += count_tokens(canon_text)

    # 3. 角色状态 (MEDIUM AUTHORITY) - 强制使用 ID 检索
    pov_id = parse_frontmatter(chapter_path)['pov']
    char_state = parse_frontmatter(f"characters/{pov_id}.md")
    state_text = f"[STATE MEMORY | MEDIUM AUTHORITY]\n{char_state}"
    context.append({"role": "system", "content": state_text})
    current_tokens += count_tokens(state_text)

    # 4. 潜意识 (LOW AUTHORITY) - 剩余预算全给潜意识
    sub_budget = BUDGET - current_tokens - 1000 # 留 1000 给近期正文
    sub = vector_retriever.query(current_text, index="subconscious", top_k=2)
    if sub:
        sub_text = f"[SUBCONSCIOUS FRAGMENT | LOW AUTHORITY | OPTIONAL]\n{sub}"
        # 如果超预算，截断潜意识
        if count_tokens(sub_text) > sub_budget:
            sub_text = truncate(sub_text, sub_budget)
        context.append({"role": "system", "content": sub_text})

    # 5. 近期正文
    context.append({"role": "user", "content": f"[Previous Text]\n{current_text}\n\nCONTINUE:"})
    
    return context
```

## 五、 CLI 命令矩阵 (V1.0.1)

| 命令 | 作用 | 防爆机制 |
| :--- | :--- | :--- |
| `loom init` | 初始化小说目录 | 生成标准 ID 模板 |
| `loom write` | Actor 交互循环 | 上下文权威分级注入 |
| `loom stash` | 存入潜意识池 | 自动增量索引 |
| `loom commit` | 提取状态并固化 | **强制 Diff Review + 前置快照** |
| `loom rollback` | 回滚错误 commit | 读取 Snapshot 恢复 YAML/SQL |
| `loom diff` | 检查正文与 Shadow 一致性 | 暴露“写伤没写治”的逻辑漏洞 |
| `loom doctor` | 诊断世界线健康度 | 检查孤立线程/时间线冲突 (钩子) |

### `loom commit` 严格审阅流程：
1.  **生成快照**：记录当前 YAML 和 SQL 状态。
2.  **Auditor 提取**：LLM 分析正文，输出 JSON 事件。
3.  **Diff 展示**：在终端用 Rich 高亮显示即将发生的变更：
    ```diff
    + [Event] char_001 获得物品: 生锈的怀表
    + [Event] char_001 状态: 左臂骨折 (Causal Pressure: 0.9)
    - [Event] char_001 情绪: 恐惧 (0.8) -> 愤怒 (0.6)
    ```
4.  **人工守门**：`Apply these changes? [y/n/edit]` (支持 e 进入 vim 手动微调 JSON)。
5.  **写入固化**：确认后，更新 Frontmatter 和 SQLite。
