## 5. 新旧架构对比

| 维度 | 旧架构 (VectorStore) | 新架构 (FTS5 管道) |
|------|---------------------|--------------------|
| 索引粒度 | 整文件级 | 段落级 (chunk) |
| 中文检索方式 | 语义相似度 | 精确关键词匹配 |
| 专有名词精确度 | 低（语义漂移风险，"影渊森林"→"神秘森林"） | 高（逐字精确匹配） |
| 角色名搜索 | 依赖向量语义 | 精确匹配"赵恒" |
| 多通道融合 | 简单拼接 | RRF 融合 (+权重) |
| 事件通道 | 无 | EventStore ×1.5 加权 |
| 增量更新 | 不支持 | FTS5 实时增量 |
| 脏数据容错 | 无兜底 | Reranker 精排兜底 |

## 9. 测试中发现并修复的问题

| 问题 | 根因 | 修复 |
|------|------|------|
| FTS5 中文搜索无结果 | unicode61 不自动拆分 CJK 连续字符 | `_cjk_space()` 逐字插入空格 |
| BM25 函数语法错误 | `rank.bm25()` 非标准 FTS5 API | 改为 `bm25(chunks_fts)` |
| 空查询导致 SQLite 报错 | 未在空查询时提前返回 | 添加上游检查 |
| chunk 过碎 | 句子边界正则含 `\n` | 移除 `\n`，段落边界已处理 |

---

## 总结

**优化的 RAG 管道在 demo_novel 项目上验证通过。**

核心改进:
1. **精准性**: FTS5 精确关键词匹配，角色名/专有名词/设定概念零语义漂移
2. **多通道互补**: FTS5(关键词) + VectorStore(语义) + EventStore(事件)，RRF 融合排序
3. **检索速度**: 0.1ms，纯本地无网络依赖
4. **增量更新**: commit/stash 时实时更新 FTS5，不阻塞创作流程
