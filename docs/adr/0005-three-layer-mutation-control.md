# 0005 — 三层变异控制机制

深层变异采用分层控制：战略层（人类作者）设定边界，战术层（策略引擎）选择维度，执行层（成本优化器）控制 Token 成本。

## 背景

四个变异维度（叙事结构、视点与声音、因果与时间线、弧光与主题）如果完全交叉组合，会产生 3⁴ = 81 个变体。每个变体都需要 Writer 生成 + Critic 评估，Token 成本会急剧上升。需要一种机制将无序的指数级探索变为有约束的阶梯式生成。

## 决策

1. **战略层（人类作者）**：设定变异边界与最终否决权。定义哪些维度可变异、哪些核心设定不可动。通过 hint 中的结构化指令（如 `<!-- structure: flashback -->`）控制。
2. **战术层（策略引擎）**：基于创作状态诊断，每次仅选择 1-2 个最相关维度进行变异，而非全量组合。动态决定生成 1-3 个变体。
3. **执行层（成本优化器）**：模型路由（大纲用小模型，正文用大模型）、上下文裁剪（仅注入相关记忆）、轻量级预筛（先用规则/小模型筛掉低分变体，再让 Critic 深度评估）。

## 考虑过的替代方案

- **全量交叉组合**：成本不可控。81 个变体 × Writer + Critic = 灾难性的 Token 消耗。
- **固定温度采样**：当前方案（0.5/0.7/0.9），本质是同一思路的不同随机采样，不是结构性变异。
- **用户全权控制**：需要用户有叙事结构知识，门槛过高。

## 影响

- Writer 的 think 阶段需要支持结构模板注入
- AutoRunner 需要集成策略引擎（可复用 Director 的分析能力）
- 需要定义变异维度的枚举和配置接口
